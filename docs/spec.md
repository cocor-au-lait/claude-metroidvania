# メトロイドヴァニア プロトタイプ 仕様書

## 概要

2D横スクロールの探索型アクションゲーム（メトロイドヴァニア）のプロトタイプ。
プレイヤーはアビリティを獲得しながらマップを探索し、ボスを倒してクリアを目指す。

## 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Phaser 3.x |
| 言語 | TypeScript |
| ビルドツール | Vite |
| 実行環境 | ブラウザ (Web) |
| バージョン管理 | GitHub |
| パッケージ管理 | npm |

## ゲーム仕様（プロトタイプスコープ：最小限）

### プレイヤー

| 項目 | 内容 |
|------|------|
| 移動 | 左右移動、ジャンプ |
| 攻撃 | 近接攻撃（剣による斬撃） |
| HP | 3（ハート3個） |
| アビリティ | 初期なし → 二段ジャンプを1つ獲得 |

**操作方法**

| キー | 動作 |
|------|------|
| ← / → または A / D | 左右移動 |
| Space または Z | ジャンプ（二段ジャンプ取得後は2回押せる） |
| X または J | 攻撃 |

### マップ構成

1エリアで構成された探索マップ。
各部屋はスクロールで繋がり、ドアや通路で行き来できる。

```
[スタート部屋]──[通路]──[アビリティ部屋]
     │                        │
  [通路]                   [通路]
     │                        │
[中間エリア]──[通路]──[ボス部屋]
```

| 部屋 | 説明 |
|------|------|
| スタート部屋 | 初期出現位置、セーブポイントあり |
| 通路・中間エリア | 敵が配置された探索エリア |
| アビリティ部屋 | 二段ジャンプを入手できる |
| ボス部屋 | ボスとの戦闘エリア（入室トリガーあり、詳細は下記） |

#### ボス部屋の入室トリガー仕様

プレイヤーがボス部屋の入口を越えた瞬間をトリガーとして、以下を同時に行う。

1. **封鎖バリアの出現** — 入口（ボス部屋左壁の開口部）に不可視の静的タイルを生成し、退路を塞ぐ
2. **ボス行動の開始** — トリガー前はボスを静止させ、トリガー後にパトロール＆ジャンプ攻撃を開始する

> **実装メモ（将来対応）**
> - トリガー検出: `GameScene` で毎フレームプレイヤー x 座標 ≥ ボス部屋左壁内側（`39 * TILE_SIZE`）を監視するか、`physics.add.overlap` でセンサー矩形を使う
> - バリア: `platforms` グループに動的タイルを追加（`group.create(x, y, "tile-boss")`）して封鎖
> - ボス制御: `Boss` に `activate()` メソッドを用意し、待機中は `setVelocityX(0)` + パトロール・ジャンプを停止する

### 敵

| 種類 | 動作 | HP | ダメージ |
|------|------|----|---------|
| スライム（雑魚） | 左右に巡回、壁で折り返す | 2 | 1 |
| ボス | **入室トリガー後に**左右移動 + ジャンプ攻撃を開始（待機中は静止） | 10 | 2 |

### アビリティ

| アビリティ | 入手場所 | 効果 |
|-----------|---------|------|
| 二段ジャンプ | アビリティ部屋 | 空中でもう一度ジャンプできる |

### セーブ・チェックポイント

- スタート部屋にセーブポイントが1か所
- 触れると HP 全回復 + リスポーン地点を記録
- ゲームオーバー時はセーブポイントから再開

### ゲームクリア・ゲームオーバー

| 条件 | 結果 |
|------|------|
| ボスを倒す | クリア画面を表示 |
| HP が 0 になる | ゲームオーバー画面 → セーブポイントから再開 |

## 参考・制約

- プロトタイプのためグラフィックは仮素材（矩形や色付きスプライト）で可
- サウンドは省略可
- タイルマップはコードで生成する（Tiled エディタ不使用）
- モバイル対応は対象外
